# The 6 Primitives

These six documents define the foundational practices for AI-native software development.
Every persona kit in this framework builds on one or more of these primitives.

| # | Primitive | Description |
|---|-----------|-------------|
| 01 | [Context Engineering](./01-context-engineering.md) | Structured context artifacts (Tier 1/2/3) that guide agents |
| 02 | [Model Routing](./02-model-routing.md) | Right model, right task, right SDLC phase |
| 03 | [Spec-Driven Development](./03-sdd-spec-driven.md) | Spec as the primary artifact; Constitution, Specification, Plan |
| 04 | [Skills and Slash Commands](./04-skills-slash-commands.md) | Reusable, governed agent skills and prompt files |
| 05 | [Quality Gates and Hooks](./05-quality-gates-hooks.md) | Automated checkpoints and agent lifecycle hooks |
| 06 | [Multi-Agent Orchestration](./06-multi-agent-orchestration.md) | Topologies for coordinating specialized agents |

## Structure

Each primitive follows a consistent format:

- **WHAT**: Definition and core concepts
- **WHY**: Evidence from academic research (arXiv papers)
- **HOW**: Step-by-step implementation with templates
- **WHO**: Responsibility matrix
- **WHEN**: Cadence and lifecycle
- **WHICH MODEL**: Model routing for each activity
- **ANTI-PATTERNS**: Common mistakes to avoid

## Research foundation

All primitives are grounded in peer-reviewed research. Key papers cited:

- 2602.20478: Context engineering for large-scale agent systems
- 2505.01622: Extended thinking impact on iterative tasks

- 2603.29919: Skill quality analysis (55K+ skills)

- 2602.16873: Multi-agent orchestration topologies
